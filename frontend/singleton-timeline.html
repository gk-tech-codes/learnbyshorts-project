üíª Sharma Ji in Code

Java Implementation

SharmaJi.java Thread-Safe 
public class SharmaJi {
    // The single instance of SharmaJi
    private static SharmaJi instance;
    
    // App configurations
    private final Map<String, String> configurations;
    
    // Private constructor prevents instantiation from outside
    private SharmaJi() {
        System.out.println("Namaste! SharmaJi is being created!");
        configurations = new HashMap<>();
        loadDefaultConfigurations();
    }
    
    // The global access point to get the same SharmaJi instance
    public static synchronized SharmaJi getInstance() {
        // Lazy initialization
        if (instance == null) {
            instance = new SharmaJi();
        }
        return instance;
    }
    
    private void loadDefaultConfigurations() {
        configurations.put("databaseUrl", "jdbc:mysql://localhost:3306/codekirana");
        configurations.put("apiEndpoint", "https://api.codekirana.com/v1");
        configurations.put("maxConnections", "100");
    }
    
    public String getConfiguration(String key) {
        return configurations.get(key);
    }
    
    public void setConfiguration(String key, String value) {
        configurations.put(key, value);
    }
    
    // This prevents cloning of SharmaJi
    @Override
    protected Object clone() throws CloneNotSupportedException {
        throw new CloneNotSupportedException("Arre bhai, there can be only one SharmaJi!");
    }
}

public class Main {
    public static void main(String[] args) {
        // Get the SharmaJi instance
        SharmaJi sharmaJi = SharmaJi.getInstance();
        
        // Frontend team uses SharmaJi
        String apiUrl = sharmaJi.getConfiguration("apiEndpoint");
        System.out.println("Frontend team using API: " + apiUrl);
        
        // Backend team changes a configuration
        sharmaJi.setConfiguration("maxConnections", "200");
        
        // DevOps team gets the updated configuration
        SharmaJi sameSharmaJi = SharmaJi.getInstance();
        String maxConnections = sameSharmaJi.getConfiguration("maxConnections");
        System.out.println("DevOps team sees maxConnections: " + maxConnections);

üåü Real-World Applications of Sharma Ji

‚öôÔ∏è Configuration Management

Just like Sharma Ji keeps track of everyone's configurations, real-world applications use Singletons for centralized configuration management.

üìù Logging

A single logging service throughout an application ensures consistent log formats and behaviors. All logs come from one Sharma Ji!

üîå Database Connections

Connection pools are perfect candidates for the Singleton pattern. Single pool manager prevents resource exhaustion.

‚ö†Ô∏è Caution: When Not to Use Sharma Ji

üß™ Testing Difficulties

Singletons can make unit testing challenging due to shared state between tests.

üîó Hidden Dependencies

Code that uses Singletons has hidden dependencies that aren't obvious.
